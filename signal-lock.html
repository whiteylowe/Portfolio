Design Intent

Signal Lock treats the tactical interface as a spatial and perceptual problem — not a stylistic one.

Built as a self-contained UI artifact, Signal Lock explores how information hierarchy, spatial framing, and motion behavior shape operator perception under high-stakes conditions. The aesthetic vocabulary is drawn from defense and surveillance systems, but the underlying questions are the same ones that drove the AR interface work at Meta: how does an interface direct attention, communicate confidence, and reduce cognitive load at the moment of decision?

Design Process
01 // Constraint
Zero Dependencies
Built entirely in vanilla HTML, CSS, and Canvas API. No frameworks, no libraries. The constraint was deliberate — understanding the rendering pipeline at a low level informed every visual decision about motion timing and draw order.
02 // Hierarchy
Information Architecture
Tactical UI design is fundamentally an information hierarchy problem. Signal Lock maps data criticality to spatial position, motion behavior, and visual weight — primary targeting data at center, contextual telemetry at periphery.
03 // Motion
Perceptual Feedback
Every animated element in Signal Lock encodes state. Scan lines indicate active acquisition, reticle behavior changes on lock confirmation, peripheral data dims during high-attention moments. Motion as communication, not decoration.
04 // Connection
From HUD to AR
The perceptual questions Signal Lock explores — attention direction, confidence signaling, ambient feedback — are the same ones that drove the AR wearable interface work at Meta Reality Labs. Different medium, same underlying design problem.
